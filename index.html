Three patches, zero encrypted files touched

A tiny Python module hooks into the AICS loader and overrides the master switches. No modification to proprietary code.

Patch 1

Force shared state

Sets has_quota=True and is_pro=True on the class-level cache that controls all AI feature gates.

Patch 2

Override PRO auth

Wraps pro_auth() to always report a valid PRO license — even when the aaPanel auth server is unreachable.

Patch 3

Rewrite server config response

Intercepts get_config() after the server response, forces quota and PRO flags back on, and downgrades PRO-only models to free-tier so they appear in your model list.

Cut out the middleman

One command to install

Works on any aaPanel installation with the AI Assistant plugin. Zero configuration needed.

1

Clone the repo

git clone https://github.com/trvanthanhhmaster-spec/aaai-free.git

2

Run the installer

sudo bash install.sh

3

Restart aaPanel

/etc/init.d/bt restart

4

Use your own API keys

Add custom accounts in AI Assistant settings. Done.

To remove: sudo bash install.sh --uninstall
